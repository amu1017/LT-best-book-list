## 推奨文

- Docker&Kubernetes ネットワークのしくみ
k8sを触り始めたけど、ネットワーク周りがモヤモヤする…という方に強くおすすめです。  
読んでまず驚いたのは、Linuxに「Network Namespace」というOS内で通信を分離できる機能があること。これがコンテナのネットワーク分離の核となっています。  
この本の特徴は、ネットワーク図が非常に直感的なこと。インターフェースとサブネットなどの空間が色分けされており、初めて聞くワードでも「どの範囲の話なのか」がイメージしやすい構成です。  
Dockerのbridgeネットワークから始まり、k8s、CNI、サービスメッシュまで段階的に解説されるので、「動くけどなぜ動くかわからない」を解消できます。

- Kubernetesの知識地図
k8sの基礎は学んだけど、「実際の現場ではどう使われているの？」という疑問を持つ方におすすめです。  
この本は全体的に現場目線で書かれており、Gitリポジトリの運用方法や、設計時にどういった観点で考えるべきかなど、実践的な内容が詰まっています。  
また、k8sエコシステムで有名なOSS（ArgoCDやCert-Managerなど）の活用方法も紹介されており、自分が知らなかったツールとの出会いもありました。教科書だけでは得られない「現場の知恵」を学べる一冊です。
